What is CSS and it is used for ?.


CSS stands for Cascading Style Sheets. CSS describes how HTML elements are to be displayed on screen, paper, or in other media CSS saves a lot of work. It can control the layout of multiple web pages all at once .External stylesheets are stored in CSS files

What is Rules, Declarations and Properties in CSS ?.


A CSS rule-set consists of a selector and a declaration block. The selector points to the HTML element you want to style.Each declaration includes a CSS property name and a value, separated by a colon. The declaration block contains one or more declarations separated by semicolons. A CSS declaration always ends with a semicolon, and declaration blocks are surrounded by curly braces.

6. What are Box Model, margins, borders and paddings?

The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.
Padding - Clears an area around the content of the box. The padding is transparent
Border - A border that goes around the padding and content of the box
Margin - Clears an area outside the border. The margin is transparent

7. What is a FlexBox?

The Flexible Box Module, usually referred to as flexbox, was designed as a one-dimensional layout model, and as a method that
could offer space distribution between items in an interface and powerful alignment capabilities. Aims at providing a more efficient way to lay out, align and distribute space among items in a container, even when their size is unknown and/or dynamic (thus the word "flex"). The main idea behind the flex layout is to give the container the ability to alter its items' width/height (and order) to best fill the available space.
A flex container expands items to fill available free space, or shrinks them to prevent overflow.
Most importantly, the flexbox layout is direction-agnostic as opposed to the regular layouts.
While those work well for pages, they lack flexibility (no pun intended) to support large or complex applications

Clean Code

1. What is refactoring?

'Refactoring' is the process of changing the syntax of a piece of code. The syntax is the spelling and grammar of a programming language. Your code is written in specific ways, with specific punctuation and rules, in order for the program to be able to carry out its function. For example, a function can be written like so: 

function exampleCode() {                         

        };
However, it can also be written like so: 
exampleCode = () => {                            
                       
        };
Both do the same, but follow different writing rules.

Changing the syntax can shorten a piece of code, reducing clutter and increasing readability.

2. What is indentation?

'Indentation' is the placement of an empty space in order to shift the position of a line. The purpose of this is to increase readability by separating lines that are above and below each other, creating paragraphs.

3. What is meant by DRY, KISS, and Separation of Concerns?

'DRY' is an acronym for 'Don't Repeat Yourself'. In every case if it is possible to not repeat code, while maintaining the functionality of your code, you should.
'KISS' is an acronym for 'Keep It Simple, Stupid'. Writing complex code can be fun to write, but a pain when you have to read and understand it. Make your code as simple as it can be.
The 'Separation of Concerns' is about separating your code into sections. Each of your sections would deal with a different 'concern'.

All of these ideas are principles that are important to keep in mind when writing your code. In the end it is all about not wasting your own, and others, time.
