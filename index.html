Clean Code

1. What is refactoring?

'Refactoring' is the process of changing the syntax of a piece of code. The syntax is the spelling and grammar of a programming language. Your code is written in specific ways, with specific punctuation and rules, in order for the program to be able to carry out its function. For example, a function can be written like so: 

function exampleCode() {                         

        };
However, it can also be written like so: 
exampleCode = () => {                            
                       
        };
Both do the same, but follow different writing rules.

Changing the syntax can shorten a piece of code, reducing clutter and increasing readability.

2. What is indentation?

'Indentation' is the placement of an empty space in order to shift the position of a line. The purpose of this is to increase readability by separating lines that are above and below each other, creating paragraphs.

3. What is meant by DRY, KISS, and Separation of Concerns?

'DRY' is an acronym for 'Don't Repeat Yourself'. In every case if it is possible to not repeat code, while maintaining the functionality of your code, you should.
'KISS' is an acronym for 'Keep It Simple, Stupid'. Writing complex code can be fun to write, but a pain when you have to read and understand it. Make your code as simple as it can be.
The 'Separation of Concerns' is about separating your code into sections. Each of your sections would deal with a different 'concern'.

All of these ideas are principles that are important to keep in mind when writing your code. In the end it is all about not wasting your own, and others, time.
